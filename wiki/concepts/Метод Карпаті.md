# Метод Карпаті

Підхід до побудови персональних баз знань з LLM, запропонований Andrej Karpathy.

## Ключова ідея
Замість RAG (перечитування сирих документів щоразу) — LLM **інкрементально будує wiki**. Знання компілюються один раз і оновлюються, а не виводяться заново на кожен запит.

## Три шари
1. **Raw** — сирі джерела (незмінні, джерело правди)
2. **Wiki** — структуровані markdown-файли (LLM пише, людина читає)
3. **Schema** — конвенції та правила (ко-еволюціонує)

## Переваги над RAG
- Синтез вже зроблений — не треба щоразу збирати з фрагментів
- Суперечності вже позначені
- Перехресні посилання вже є
- Масштабується до ~500 статей без vector search

## Інструменти
- [[Obsidian]] — IDE для wiki (граф, пошук, теги)
- `index.md` — каталог для LLM-навігації
- `log.md` — хронологічний журнал
- `qmd` — пошук при масштабуванні (BM25 + vector)

## Адаптація в нашій системі

- Компіляція через cron job [[OpenClaw]]
