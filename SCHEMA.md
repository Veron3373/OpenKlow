# CoreVault — Персональна база знань

## Архітектура

CoreVault побудований за методом Карпаті (Andrej Karpathy): LLM інкрементально будує та підтримує структуровану wiki замість щоразового перечитування сирих документів.

## Три шари

### 1. Raw (`raw/`)
Сирі джерела — статті, нотатки, документи, зображення. **Незмінні** — Адам читає, але ніколи не змінює. Це джерело правди.
- `raw/assets/` — зображення, PDF, медіа

### 2. Wiki (`wiki/`)
Структурована wiki з markdown-файлів. **Адам пише і підтримує цей шар повністю.** Сторінки сутностей, концепцій, підсумків, порівнянь. Алім читає; Адам пише.
- `wiki/entities/` — сторінки сутностей (люди, проєкти, інструменти)
- `wiki/concepts/` — концепції, теми, патерни
- `wiki/sources/` — підсумки опрацьованих джерел
- `wiki/index.md` — каталог всіх сторінок
- `wiki/log.md` — хронологічний журнал дій

### 3. Схема (цей файл)
Конвенції, структура, воркфлоу. Ко-еволюціонує з часом.

## Воркфлоу

### Ingest (обробка нового джерела)
1. Нове джерело з'являється в `raw/`
2. Адам читає, обговорює ключові тези з Алімом
3. Створює summary-сторінку в `wiki/sources/`
4. Оновлює відповідні entity/concept сторінки
5. Оновлює `wiki/index.md`
6. Додає запис в `wiki/log.md`

### Query (запит до бази)
1. Адам читає `wiki/index.md` для навігації
2. Знаходить релевантні сторінки
3. Синтезує відповідь з цитатами
4. Цінні відповіді зберігаються як нові wiki-сторінки

### Lint (перевірка здоров'я)
Періодично Адам перевіряє:
- Суперечності між сторінками
- Застарілі твердження
- Сирітські сторінки без вхідних посилань
- Згадані але не створені концепції
- Пропущені перехресні посилання

## Конвенції

### Формат сторінок
```markdown
---
tags: [тег1, тег2]
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources: [назва-джерела]
---

# Назва

Зміст з [[посиланнями]] на інші сторінки.
```

### Посилання
- Використовувати `[[wikilinks]]` для зв'язків між сторінками
- Кожна сторінка має мати хоча б одне вхідне посилання

### Мова
- Українська (основна)
- Англійська для технічних термінів де доречно

## Автокомпіляція пам'яті

OpenClaw автоматично компілює щоденні логи (`memory/YYYY-MM-DD.md`) у структуровані wiki-статті.

### Цикл:
1. Адам веде щоденні логи в `memory/`
2. Cron job (щодня о 22:00 Kyiv) запускає компіляцію
3. Компілятор переглядає нові записи за день
4. Витягує: рішення, уроки, патерни, помилки, інсайти
5. Створює/оновлює сторінки в `wiki/concepts/`, `wiki/entities/`
6. Оновлює `wiki/index.md` та `wiki/log.md`

### Категорії знань:
- **concepts/** — патерни, підходи, архітектурні рішення
- **entities/** — люди, проєкти, інструменти, сервіси
- **sources/** — підсумки зовнішніх джерел
- **connections/** — зв'язки між концепціями
- **qa/** — важливі Q&A що варто зберегти

### Lint (перевірка здоров'я)
Періодично (раз на тиждень) Адам перевіряє:
1. Зламані `[[посилання]]`
2. Сирітські сторінки (без вхідних посилань)
3. Суперечності між сторінками
4. Застарілі дані
5. Згадані але не створені концепції
6. Пропущені перехресні посилання
7. Порожні або занадто короткі сторінки

## Інструменти
- **Obsidian** — UI для перегляду wiki, граф зв'язків
- **Obsidian Web Clipper** — збереження статей у raw/
- **Graph View** — візуалізація зв'язків
